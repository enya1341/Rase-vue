# rase-vue (フロントエンド)

<br>


## ※一覧は基本ファイル順

<br>


> ### 使用言語

<br>


- vue.js

  - HTML＆CSS

  - JavaScript

<br>

> ### サーバー

<br>

- S3 [rase-enya](http://rase-enya.s3-website-ap-northeast-1.amazonaws.com)

<br>

> ### 環境変数

<br>

- VUE_APP_LOCAL_APIKEY  *(localや開発環境でのfirebaseのAPIKEY)*

- VUE_APP_NOTLOCAL_APIKEY *(本番環境でのfirebaseのAPIKEY)*

- NODE_ENV *(環境を識別するためのNODE_ENV)*

- VUE_APP_API_BASE_URL *(環境ごとのapiのURL)*

<br>

> ### 追加案件

<br>

 - ER図 (ER図を作成する)

 - 予約変更機能 (予約日時または予約人数をマイページから変更することができる)

 - 評価機能 (予約したお店に来店した後に、利用者が店舗を5段階評価できるようにする)

 - バリデーション (予約と認証の際にフロントエンドとバックエンドそれぞれバリデーションをかける)

 - 管理画面 (管理者と店舗代表者と利用者の3つの権限を作成する。店舗代表者が店舗情報の作成、更新と予約情報の確認ができる管理画面を作成する。管理者側は店舗代表者を作成できる管理画面を作成する)

 - 認証 (Firebase Authenticationを利用して認証の仕組みを導入する。認証の有無によってナビゲーションガードを使用して弾く)

 - ストレージ (お店の画像をストレージ（S3）に保存することができる)

 - QRコード (利用者が来店した際に店舗側に見せるQRコードを発行し、お店側は照合することができる)

 - テストコード (PHPUnitを使用してAPIのテストを作成する)

 - Docker (Dockerを使った環境を構築する)

 - AWS(S3のみ)

 - 環境の切り分け (開発環境と本番環境の切り分けを行う。それぞれに合わせて環境変数を設定する)
